Ask your Question DBA web-based CRUD (Create, Read, Update, Delete) application designed as a question-sharing platform for students. This system allows students to interact in a secure environment while providing administrators with full system management capabilities.

🚀 FeaturesPublic InterfaceView-Only Access: Unauthenticated users can view the list of questions and see system statistics (number of questions, registered users, and available modules) .Navigation Restrictions: Users must log in to perform any modifications or post content.Student InterfaceQuestion Management: Registered students can create new questions, view their own posts, and edit or delete their content.Module Categorization: Questions are assigned to specific academic modules via a dynamic drop-down menu.Communication: Includes a "Contact Us" facility to send messages directly to administrators.Admin InterfaceUser & Module Management: Administrators can add, edit, or delete users and academic modules.Content Moderation: Elevated privileges to manage all questions across the platform to ensure academic integrity.

🛠️ Technology StackFrontend: HTML5 and CSS for a semantic and user-friendly interface.Backend: PHP using PDO (PHP Data Objects) to prevent SQL injection through prepared statements .Database: MySQL relational database architecture with referential integrity.Accessibility: Designed following WCAG 2.1 (AA standard) for color contrast and readability.

📊 Data ModelingThe system utilizes a relational schema with three primary entities:Users: Stores credentials and roles (Admin/Student).Questions: Linked to authors and modules via Foreign Keys.Modules: Categories for academic subjects.

🛡️ Security & Legal ComplianceUK GDPR: Adheres to data minimization principles (only collecting necessary usernames/passwords) and supports the "Right to Erasure".SQL Injection Protection: All database interactions use PDO prepared statements.Accountability: Every post is linked to a specific user account to discourage abusive behavior or academic misconduct .

🧪 Testing OverviewThe application underwent structured Manual Black-Box Testing:Functional Testing: Verified core CRUD operations and authentication.Negative Testing: Tested error handling for invalid credentials and empty form submissions.Security Testing: Confirmed the prevention of SQL injections.Note: Current known limitations include a need for improved session-based access control (TC-11) and automated server-side image cleanup (TC-24)
